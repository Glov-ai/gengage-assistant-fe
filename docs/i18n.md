# i18n And Localization

The SDK ships built-in Turkish and English strings for all user-facing widgets. Locale handling is intentionally lightweight:

- pass `locale` to pick a built-in string set
- pass `i18n` to override any subset of strings
- fork the relevant locale module when you need a fully new locale

## Widget Locale Surface

| Widget | Built-In Locale Module | Config Fields | Notes |
|--------|------------------------|---------------|-------|
| Chat | `src/chat/locales/` | `locale`, `i18n?: Partial<ChatI18n>` | Also maps locale to backend output language in `src/chat/kvkk.ts` |
| QNA | `src/qna/locales/` | `locale`, `i18n?: Partial<QNAI18n>` | Locale controls launcher labels and fallback input copy |
| SimRel | `src/simrel/locales/` | `locale`, `i18n?: Partial<SimRelI18n>` | Locale controls grid labels, CTA text, retry copy |
| SimBut | `src/simbut/locales.ts` | `locale`, `i18n?: Partial<SimButI18n>` | Only the find-similar pill label is localized |

## Locale Resolution Rules

- Chat, QNA, and SimRel normalize locale strings in their `locales/index.ts` helpers.
- SimBut resolves English when `locale` starts with `en`; all other values fall back to Turkish.
- Partial `i18n` overrides are merged on top of the resolved built-in locale object.

In practice, these all work:

- `locale: 'tr'`
- `locale: 'tr-TR'`
- `locale: 'en'`
- `locale: 'en-US'`

## Overlay-Level Locale

When you use `initOverlayWidgets(...)`, the shared `locale` option is forwarded into child widgets unless a widget-level override replaces it.

```ts
await initOverlayWidgets({
  accountId: 'mystore',
  middlewareUrl: 'https://YOUR_BACKEND_URL',
  locale: 'en',
  chat: { variant: 'floating' },
  qna: { mountTarget: '#qna' },
  simrel: { mountTarget: '#simrel' },
  simbut: { mountTarget: '#gallery' },
});
```

## Partial String Overrides

Use `i18n` for targeted copy changes instead of forking locale files.

```ts
const chat = new GengageChat();
await chat.init({
  accountId: 'mystore',
  middlewareUrl: 'https://YOUR_BACKEND_URL',
  locale: 'tr',
  i18n: {
    inputPlaceholder: 'Sorunuzu yazin...',
    sendButton: 'Gonder',
  },
});
```

## Adding A New Locale In A Fork

1. Add a locale file under the relevant widget directory:
   - `src/chat/locales/`
   - `src/qna/locales/`
   - `src/simrel/locales/`
   - or extend `src/simbut/locales.ts`
2. Update the widget's locale resolver to return the new locale.
3. Keep the `i18n` interface as the single source of truth for required strings.
4. Verify the new locale in both live widgets and `npm run catalog`.

## Chat-Specific Notes

- Chat has the deepest i18n surface because it owns recovery UI, panel titles, voice labels, consulting mode strings, and attachment flows.
- Beauty and watch consulting strings are also supplied through `ChatI18n` and can be overridden without replacing renderers.
- Consulting fallback labels such as unnamed recommendation groups, fallback style-card captions, the beauty selfie step, and the photo-analysis card all resolve through `src/chat/locales/` the same way as the rest of the chat widget.
- `choicePrompter` is intentionally suppressed in beauty-consulting mode regardless of locale.
